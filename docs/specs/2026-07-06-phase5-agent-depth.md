# 解牛 Phase 5：从「资讯站」到「私人投研 Agent」的深度（loop `agentdepth` 锚点）

> 缘起：sway 分享了一份 ChatGPT 对解牛的产品批评（标题「财经Agent产品优化建议」，基于实际浏览 jieniu.swaylab.ai）。本 spec 把该批评**去重、对照解牛现状、排优先级**成可执行 backlog。

## ChatGPT 的核心判断（原文精神）
> 解牛现在是个**完成度不错的财经资讯网站，但还不像真正的财经 Agent**。核心体验仍是「聚合新闻→贴标签→用户点 AI 解读」，和东方财富/财联社/华尔街见闻无本质区别。
> **应把产品中心从「新闻」换成「用户正在形成和维护的投资判断（thesis）」。** 不只告诉你发生了什么，而是告诉你这件事**是否改变了你的投资判断**。

## ★ 关键洞见：解牛其实已经走在这条路上（Phase 3/4 的验证）
ChatGPT 的很多「建议」解牛**已经做了**——这份批评反而**独立印证了 P3/P4 的方向**：

| ChatGPT 建议 | 解牛现状 |
|---|---|
| 首页=今日投资变化，不是新闻流 | ✅「今天你的组合变了什么」PortfolioChanged + 传播链 PortfolioImpact |
| 事件去重/合并 | ✅ `event-cluster.ts` + NewsEvent 合并（同题 N 篇） |
| 公司页=持续更新的 investment memo，维度追踪 | ✅ 「投资逻辑·X」卡：盯这几个维度 + 兑现/风险 + 维度状态 bullish/bearish |
| 关注=记忆系统（why/持仓/成本/目标/风险/替代） | ✅ 四层 Memory：User 画像 / Portfolio 持仓成本仓位目标 / Thesis 含催化+证伪 / Decision 决策史 |
| 大师视角降级为可选镜头，默认=事实→预期→超预期→影响→逻辑增强削弱 | ⚠️ 已有 thesisLens（thesis 相对解读）+ 通用 AI + master-compass；需确认默认=thesis 解读、大师 opt-in |
| 「深度分析必须窄」不要堆全市场 | ✅ 省 token 铁律：AI 只给 持仓/热门股/thesis-with-changes；501 公司只是**数据广**、分析窄——**正是 ChatGPT 要的「覆盖广、深度窄」** |
| Agent 六类用户状态 + 五层公司知识 | ✅ 基本对应四层 Memory + 实体/信号/维度/命题 |

**结论**：不推倒重来。补的是 ChatGPT 点到、解牛还缺的**深度增量**（下方 backlog）。

## 铁律（延续）
颜色红/绿只给真实价格涨跌，thesis 方向/状态/新信息程度一律 amber/灰；合规不荐股/不点位/不承诺收益；AI/Code 边界（数字必来自 DB，不编）；**省 token**（AI 只给 持仓/热门股/thesis 变化；去重/归因/排序/新信息程度/机会规则全部纯规则零 AI）。
